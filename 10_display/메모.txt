css사용시 불특정태그 사용 시, 컴마로 연결
*은 html에 있는 모든 태그 => html의 바디 영역

화면상에 보이는 태그 유형(display속성)
display유형의 종류와 특성

1.block(div,hn,p,ul,ol,등       // li)
기본사이즈: 가로 - 100% (브라우저 크기에 따라 변함), 세로 - 태그의 자식이 존재하는 만큼
장점: 사이즈 조정이 가능하다 (width/height)
단점: 세로로 배치된다 (홈페이지 내용이 길어짐->사용자가 보기 불편해서 가로배치로 바꿔줘야함) 
	  =>★float를 사용하여 변경 /
	      최근엔 플랙스 사용 (익스플로어 사용으로 과거엔 사용하지 않음)

2.inline(a, span)
기본사이즈: 가로 세로 모두 자식이 존재하는 만큼
장점: 가로로 배치된다
단점: 사이즈 조정이 불가능하다

3. inline-block(img,input)
기본사이즈: 가로 세로 모두 자신이 존재하는 만큼 (img태그는 단독태그기에 자식이 없음)
장점: 가로로 배치된다 + 사이즈 조절이 가능하다
단점: 공백해결이 불편하다
**클릭이 안되는 이미지들은 background image // 이미지 태그아님


=> block안에 inline있어야함!!!!!!!!!! <h><a href=""></a></h>
   display유형을 강제로 바꿀 수 있으나 권장하지 않음
=> img태그는 기본적으로
   가로 width : 자신이 존재하는 만큼 / 세로 height : auto로 생성된다 (자동으로 맞춰지는 것)
   height:auto 라는 것은 -> 이미지의 가로 비율에 맞춰 세로 값을 자동으로 만든다
   ★img태그에는 height 값을 작성하지 않는다★

=>html의 크기 단위에는 px과 %가 대표적이다
  ※%의 기준은 부모의 크기이다
  할아버지 = 1000px ,  아버지 = 50% , 나 = 50% -> 250px
