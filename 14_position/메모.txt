top/left/right/bottom
->position을 활용하여 이동하는 것
※사용하려면 필수적으로 position이 필요하다

position -> 이동하는 방법을 지시

[relative] margin-top이 상속되어 사용 불가능한 경우
1. 상대좌표에 변화가 없다(자신의 자리를 지킨다)
    + 규칙을 지킨다 (margin:0 auto와 함께 사용가능)
2. z-index:auto가 다른 z-index:auto로 변경되나
   top/left/right/bottom을 사용하지 않는다면 동생이 형을 업지 않는다.
   (남을 무시하지 않는다 / 성적표가 공개되면 남을 무시한다)
3. 상대좌표를 기준으로 top/left/right/bottom이 적용된다
   (자신의 자리를 기준점으로 이동한다)


[absolute] 형제관계를 겹칠 때
1. 상대좌표에 변화가 있다(자신의 자리를 지키지 않는다)
   ※ 부모의 가로 초기점으로 이동한다
    + 규칙을 지키지 않는다 (margin:0 auto와 함께 사용 불가능)
2. z-index:auto가 다른 z-index:auto로 변경된다
   (동생태그가 형태그의 위치를 침범함)
3. 절대좌표(브라우저 초기값)를 기준으로 top/left/right/bottom이 적용된다
   (브라우저의 크기에 따라 위치가 변경된다)
    ->자리를 쉽게 배치하려고 absolute만 활용하면, 
    기기가 달라졌을 때(PC, 스마트폰, 등) 배치가 전부 깨짐 => 방향성을 넣을 때 absolute 넣지 않는 것을 추천!!
4. 부모가 position을 가지고 있다면 (relative) top/left/right/bottom을 사용하더라도
   부모태그 안에서만(부모태그의 초기점에서) 이동된다
   * but, 부모태그가 absolute나 없으면 브라우저 초기값이 기준됨

★★★ 무조건 absolute가 부여된 태그의 부모에게 height값을 부여해야함 -> 자식태그 높이가 줄어들면 부모태그 높이값이 함꼐 줄어들기 때문 
★★★ 상황별 top/left/right/bottom을 사용하면 부모에게 position을 부여해야함


[fixed] -> 고정되어 스크롤을 따라다닐 떄 사용
1. 상대좌표에 변화가 있다(자신의 자리를 지키지 않는다)
   ※ 부모의 가로 초기점으로 이동한다
    + 규칙을 지키지 않는다 (margin:0 auto와 함께 사용 불가능)
2. z-index:auto가 다른 z-index:auto로 변경된다
   (동생태그가 형태그의 위치를 침범함)
3. 절대좌표(브라우저 초기값)를 기준으로 top/left/right/bottom이 적용된다
   (브라우저의 크기에 따라 위치가 변경된다)

★★★ 부모를 배치하고 자신을 fixed적용 -> 정보를 가리지 않는다 (무)
★★★ 자신을 배치하고 fixed 적용 -> 정보를 가린다 (유)
        => 자기 자신에게 top/left/right/bottom 유무차이
